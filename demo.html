jQuery.gridifier

Gridifier is a lightweight jQuery plugin whose only purpose is to fit your images into nifty boxes.
It even provides some options for you to customize its behaviour.

Gridifier was made to be used in real websites, so it's optimized for the web, it's flexible and it works!

Basic usage

It doesn't take much to implement Gridifier. Just load the script in your page: 

<script src="jquery.gridify.js"></script>

Then choose your target containers (the elements containing the images to resize). The container should have at least one CSS property defined among width, height, max-width and max-height (or else it'll stretch to the image's size).
Of course you can experiment, and even tweak Gridifier's CSS and script to fit your own needs!
These are, however, the three base models you can start using right away: 

<container>
	<img />
</container>
		
<container>
	<a>
		<img />
	</a>
</container>
		
<container>
	<img />
	<cover>
		<text>
		</text>
	</cover>
</container>

NOTE: container, cover and text are, of course, meta-language, meaning that they're not real tags.
You can use anything you want as a container for your image, best practices are using div, li tags, or the new HTML5 figure tag.
The container is identified by the CSS default with the container class, but you can change that.

Img is, of course, the image you want to fit in the container.

You can wrap that img in an anchor, if you want it to be clickable (see example 2).

The cover is a cover for the element, designed to slide over the image on hover, to display informations (like a title for the image), and optionally to provide a clickable link, using an anchor as the cover.
This element is identified with the grdCover class, but you can change that (you'd have to change both the .css and .js files).

finally, call the .gridify() function in your pages on window.load event: 

<script>
	$(window).load( function(){
		$(".container").gridify();
	});
</script>

NOTE: Why the window.load event?

When a page is loading, it launches a series of events that indicate the loading progress. The first of these is document.ready (the well-known $() or jQuery() ), which means that the structure of the page has been loaded.
That structure is the DOM (Document Object Model), the tree of the page's elements. When the DOM is loaded, the images have been loaded as nodes of the DOM, but they haven't been actually loaded as images.

For the proper functioning of Gridifier we need the images to be fully charged, and this is true when the page triggers the window.load event.

Settings

1. forceFit, which forces Gridifier into upscaling images which are smaller than their container.
This feature is disabled by default, for not everyone likes their images to be upscaled (image quality is decreased this way).

To enable forceFit use: 

$(window).load( function(){
	$(".container").gridify({
		forceFit: true
	});
});

2. debug, which outputs the before and after of the images in your browser's console, making it easy for you to debug the script.

To enable debug use: 

$(window).load( function(){
	$(".container").gridify({
		debug: true
	});
});

3. align, which lets you choose how to align the image inside the container.
The only possible values are left, center, right for horizontal alignment, and top, center, bottom for vertical alignment.

If the container's width and height are actually gonna vary (maybe on window resize, or other events), you should call the .gridify() function again.
You can do that by using this additional piece of code (don't remove the window.load part!) : 

$(window).resize( function(){
	$(".grid img").gridify();
});
